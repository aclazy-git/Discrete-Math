# 获取电子档

https://zh.zlib.li/

在 zlib 上获取电子书。

# 得到初稿

https://mineru.net/OpenSourceTools/Extractor

通过 mineru 将电子书初步解析成 markdown 文件。（因为单次解析对文件大小以及页数有要求，所以将 pdf 拆成五份进行解析，对应 part 1,2,3,4,5）

解析后产生了以下文件
1. full.md
2. image/
3. *.json

full.md 和 image/ 则是我们所需要的初稿。保留 .json文件 目的是方便让你将该文件夹导入到 mineru 再次进行格式转换，比如 latex,json,html 等。
# 得到源稿

因为 OCR 工具的局限，初稿难免会有格式上的瑕疵。

在学习的同时，调整初稿的格式，得到最重要的 **源稿** 。

值得注意的是，对待源稿，一定要遵循 **忠于原文** 的原则，在最大程度上复制原 PDF 的内容。

为什么呢？

实际上，每个人对内容的理解方式不同，语言习惯也不同，对同一本书输出的笔记差异极大。很普遍的是，别人的笔记对你自己而言，可读性并不强。为此，保留一个原文内容未经污染的 **源稿** 极其重要。

那么如何插入自己的注释或者笔记或者视频呢？

我们作以下约定：

1. 通过 %%作者名：内容%% 的方式将内容插入到源稿。
2. 创建子文件夹来存放自己的相关内容，比如 离散part1/aclazy ，再在源文件中引用你的文件。 比如 离散part1/aclazy/第一章节的理解.md 。
